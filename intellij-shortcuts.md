# Principais Atalhos de Teclado do IntelliJ IDEA

O IntelliJ IDEA é uma IDE poderosa com diversos atalhos de teclado que podem ajudar a acelerar o fluxo de trabalho dos desenvolvedores. Abaixo estão os principais atalhos usados no desenvolvimento.

| **Ação**                                 | **Atalho (macOS)**        | **Atalho (Windows/Linux)**  |
|------------------------------------------|---------------------------|-----------------------------|
| **Busca Geral (por qualquer coisa)**     | `⇧⌘A` (Shift + Command + A)| `Ctrl + Shift + A`          |
| **Auto completar código**                | `⌃Espaço` (Ctrl + Space)   | `Ctrl + Space`              |
| **Auto completar inteligente**           | `^⇧Espaço` (Ctrl + Shift + Space)| `Ctrl + Shift + Space`  |
| **Gerar código (getters, setters, etc.)**| `⌘N` (Command + N)         | `Alt + Insert`              |
| **Reformatar código**                    | `⌥⌘L` (Option + Command + L)| `Ctrl + Alt + L`           |
| **Refatorar código**                     | `⌘T` (Command + T)         | `Ctrl + Shift + Alt + T`    |
| **Pesquisar Usos (Find Usages)**         | `⌥F7` (Option + F7)        | `Alt + F7`                  |
| **Navegar até a declaração**             | `⌘B` (Command + B)         | `Ctrl + B`                  |
| **Ir para implementação**                | `⌥⌘B` (Option + Command + B)| `Ctrl + Alt + B`           |
| **Abrir Arquivo (Go to File)**           | `⌘O` (Command + O)         | `Ctrl + N`                  |
| **Abrir Classe (Go to Class)**           | `⌘⇧O` (Command + Shift + O) | `Ctrl + Shift + N`         |
| **Ir para Linha**                        | `⌘L` (Command + L)         | `Ctrl + G`                  |
| **Executar Programa**                    | `⌃R` (Control + R)         | `Shift + F10`               |
| **Debuggar Programa**                    | `⌃D` (Control + D)         | `Shift + F9`                |
| **Adicionar Ponto de Interrupção**       | `⌘F8` (Command + F8)       | `Ctrl + F8`                 |
| **Alternar Entre Arquivos Recentes**     | `⌘E` (Command + E)         | `Ctrl + E`                  |
| **Ir para a Definição de Testes**        | `⌘⇧T` (Command + Shift + T)| `Ctrl + Shift + T`          |
| **Duplicar Linha**                       | `⌘D` (Command + D)         | `Ctrl + D`                  |
| **Excluir Linha**                        | `⌘⌫` (Command + Backspace) | `Ctrl + Y`                  |
| **Comentar/Descomentar Linha**           | `⌘/` (Command + /)         | `Ctrl + /`                  |
| **Mover Linha para Cima/Baixo**          | `⌥⇧↑` / `⌥⇧↓` (Option + Shift + Up/Down)| `Shift + Alt + Up/Down`|
| **Desfazer Ação**                        | `⌘Z` (Command + Z)         | `Ctrl + Z`                  |
| **Refazer Ação**                         | `⌘⇧Z` (Command + Shift + Z)| `Ctrl + Shift + Z`          |

## Explicação dos Principais Atalhos

- **Busca Geral (⇧⌘A / Ctrl + Shift + A)**: Abre uma caixa de busca universal onde você pode pesquisar qualquer comando ou configuração do IntelliJ.

- **Auto Completar Código (⌃Espaço / Ctrl + Space)**: Sugere possíveis conclusões para o código, como métodos ou variáveis.

- **Reformatar Código (⌥⌘L / Ctrl + Alt + L)**: Aplica a formatação padrão de código definida no projeto, corrigindo indentação e espaços em branco.

- **Refatorar Código (⌘T / Ctrl + Shift + Alt + T)**: Abre um menu de refatoração, permitindo renomear, mover, alterar a assinatura do método, etc.

- **Navegar até a Declaração (⌘B / Ctrl + B)**: Leva você até a declaração de uma variável, função ou classe onde ela foi definida.

- **Ir para Implementação (⌥⌘B / Ctrl + Alt + B)**: Leva diretamente para a implementação de uma interface ou classe abstrata.

- **Executar Programa (⌃R / Shift + F10)**: Executa o projeto ou módulo atual.

- **Debuggar Programa (⌃D / Shift + F9)**: Inicia o processo de depuração, permitindo o uso de breakpoints para inspecionar a execução.

- **Adicionar Ponto de Interrupção (⌘F8 / Ctrl + F8)**: Adiciona ou remove um breakpoint na linha atual durante a depuração.

- **Comentar/Descomentar Linha (⌘/ / Ctrl + /)**: Comenta ou remove o comentário da linha atual ou de uma seleção.

Esses atalhos são extremamente úteis para agilizar o desenvolvimento e aumentar a produtividade ao trabalhar com o IntelliJ IDEA.

## Explicação dos Principais Atalhos

- **Alternar entre Editores Recentes (⌘E / Ctrl + E)**: Abre uma lista suspensa dos arquivos editados recentemente para alternar entre eles rapidamente.

- **Alternar entre Janelas Recentes (⌘⇥ / Alt + Tab)**: Permite navegar rapidamente entre janelas de ferramenta e o editor principal.

- **Ir para a Janela de Ferramenta (⌘[número] / Alt + [número])**: Abre uma janela de ferramenta específica, como o painel de projeto (geralmente `⌘1` / `Alt + 1`) ou o terminal (`⌥F12` / `Alt + F12`).

- **Ocultar/Mostrar todas as Janelas de Ferramenta (⇧⌘F12 / Ctrl + Shift + F12)**: Oculta ou exibe todas as janelas de ferramenta para maximizar o espaço do editor.

- **Focar no Editor (Esc)**: Retorna o foco ao editor de código, fechando qualquer janela de ferramenta que esteja ativa.

- **Maximizar/Restaurar Janela de Ferramenta (Ctrl + ⇧⌘F12 / Ctrl + Shift + F12)**: Alterna entre maximizar uma janela de ferramenta e restaurá-la ao seu estado original.

- **Exibir/Esconder o Painel de Projeto (⌘1 / Alt + 1)**: Abre ou fecha o painel de projeto, onde você pode navegar pelos arquivos do projeto.

- **Alternar Entre Abas no Editor (⌃⇥ / Ctrl + Tab)**: Navega entre as abas abertas no editor sem usar o mouse.

- **Dividir Editor Horizontalmente (⌘⇧D / Ctrl + Shift + D)**: Divide a janela do editor, permitindo visualizar dois arquivos ou duas partes do mesmo arquivo ao mesmo tempo.

- **Fechar Editor Dividido (⌘⇧F4 / Ctrl + Shift + F4)**: Fecha a divisão do editor e retorna à visualização de uma única janela.

Esses atalhos ajudam a controlar a navegação entre janelas e painéis no IntelliJ IDEA de maneira eficiente, permitindo que o desenvolvedor mantenha o foco e a produtividade enquanto trabalha.

# Principais Atalhos de Run, Debug e Imports no IntelliJ IDEA

Aqui estão os principais atalhos de teclado para as ações relacionadas a **execução (Run)**, **depuração (Debug)** e **importação (Imports)** no IntelliJ IDEA, que podem agilizar bastante o desenvolvimento.

## Run (Executar)

| **Ação**                                    | **Atalho (macOS)**          | **Atalho (Windows/Linux)**  |

## Imports (Importações)

| **Ação**                                    | **Atalho (macOS)**           | **Atalho (Windows/Linux)**  |
|---------------------------------------------|------------------------------|-----------------------------|
| **Otimizar Imports**                        | `⌃⌥O` (Control + Option + O)  | `Ctrl + Alt + O`            |

| **Visualizar Caminhos de Importação**       | `⌘P` (Command + P)            | `Ctrl + P`                  |

## Explicação dos Principais Atalhos

### Run (Executar)
- **Executar Programa (⌃R / Shift + F10)**: Este é o comando principal para rodar o código. Ele executa o programa com base na configuração de execução selecionada.

- **Executar Última Configuração (⌃⌥R / Alt + Shift + F10)**: Executa a última configuração de execução usada, facilitando a repetição de testes ou execuções sem precisar selecionar novamente.

- **Recompilar Projeto (⌘F9 / Ctrl + F9)**: Recompila todo o projeto ou módulo atual sem executar o código.

- **Parar Execução (⌃F2 / Ctrl + F2)**: Para a execução atual do programa.

### Debug (Depuração)
- **Iniciar Depuração (⌃D / Shift + F9)**: Inicia o programa em modo de depuração, permitindo a inserção de breakpoints e análise da execução em tempo real.

- **Adicionar/Remover Ponto de Interrupção (⌘F8 / Ctrl + F8)**: Adiciona ou remove um breakpoint na linha atual do código, útil para parar a execução no ponto exato desejado.

- **Passo para Dentro/Fora/Sobre (F7 / Shift + F8 / F8)**: Esses comandos permitem navegar pela execução do código linha por linha. Usar `Step Into` (F7) vai entrar em métodos ou funções chamadas, enquanto `Step Over` (F8) passa por cima dessas chamadas.

- **Avaliar Expressão (⌥F8 / Alt + F8)**: Permite avaliar expressões durante a depuração, ajudando a verificar o estado das variáveis ou executar comandos diretamente no contexto da execução.

### Imports (Importações)
- **Otimizar Imports (⌃⌥O / Ctrl + Alt + O)**: Remove importações não usadas e organiza os pacotes de importação no arquivo atual.

- **Auto Importar Classes e Pacotes (⌃Espaço / Ctrl + Space)**: Sugere e completa automaticamente importações necessárias enquanto você escreve o código, útil quando você usa classes de outros pacotes.

- **Corrigir Importações (⌥Enter / Alt + Enter)**: Oferece uma sugestão rápida para corrigir importações quebradas ou ausentes no código.

- **Reformatar e Otimizar Imports (⌥⌘L / Ctrl + Alt + L)**: Não só formata o código conforme o padrão, mas também otimiza automaticamente as importações durante o processo.

Esses atalhos são extremamente úteis para desenvolver e depurar código no IntelliJ IDEA, tornando o fluxo de trabalho mais rápido e eficiente.

# Principais Atalhos de Build no IntelliJ IDEA

A seguir estão os principais atalhos relacionados ao **Build** no IntelliJ IDEA. Esses comandos são essenciais para compilar o código, gerar artefatos e executar scripts de build.

| **Ação**                                 | **Atalho (macOS)**           | **Atalho (Windows/Linux)**  |
|------------------------------------------|------------------------------|-----------------------------|
| **Compilar Arquivo Atual**               | `⌘F9` (Command + F9)          | `Ctrl + F9`                 |
| **Build do Projeto (Build All)**         | `⌘⇧F9` (Command + Shift + F9) | `Ctrl + Shift + F9`         |
| **Executar Build Limpo (Clean Build)**   | -                            | -                           |
| **Reconstruir Projeto (Rebuild Project)**| `⌘⇧F9` (Command + Shift + F9) | `Ctrl + Shift + F9`         |
| **Gerar Artefatos**                      | `⌥⌘F9` (Option + Command + F9)| `Alt + Shift + F9`          |
| **Verificar Erros de Compilação**        | `⌘K` (Command + K)            | `Ctrl + K`                  |
| **Abrir Resultados de Build**            | `⌘0` (Command + 0)            | `Alt + 0`                   |
| **Executar Build Gradle (ou Maven)**     | `⌥F12` (Option + F12)         | `Alt + F12`                 |
| **Executar Tarefas de Build Específicas (Gradle/Maven)**| `⌘⇧A` (Command + Shift + A) | `Ctrl + Shift + A`          |
| **Abrir Janela de Build**                | `⌘9` (Command + 9)            | `Alt + 9`                   |

## Explicação dos Principais Atalhos

### Compilar Arquivo Atual (⌘F9 / Ctrl + F9)
Este comando compila apenas o arquivo ou módulo que está atualmente aberto no editor. Isso permite verificar rapidamente o código para erros sem precisar compilar todo o projeto.

### Build do Projeto (⌘⇧F9 / Ctrl + Shift + F9)
Este comando compila todos os arquivos do projeto ou do módulo atual. Ele verifica a integridade do código em todo o projeto e é usado quando você precisa garantir que todas as partes do projeto estejam corretamente compiladas.

### Reconstruir Projeto (⌘⇧F9 / Ctrl + Shift + F9)
Reconstrói o projeto completamente, limpando todos os arquivos compilados anteriormente e gerando novos. Este comando é útil quando você precisa de uma compilação limpa após alterar arquivos de configuração, dependências ou quando enfrentar problemas de cache.

### Gerar Artefatos (⌥⌘F9 / Alt + Shift + F9)
Este comando gera os artefatos configurados no projeto, como pacotes .jar, .war ou outros binários de distribuição. Você pode configurar e gerar esses artefatos a partir das configurações do projeto.

### Verificar Erros de Compilação (⌘K / Ctrl + K)
Este comando ajuda a verificar a presença de erros de compilação no código atual e fornece um relatório dos problemas encontrados, facilitando a correção antes de executar um build completo.

### Abrir Resultados de Build (⌘0 / Alt + 0)
Este atalho abre a janela de resultados do build, onde você pode visualizar os logs e detalhes sobre a execução da compilação, erros e advertências que ocorreram durante o processo.

### Executar Build Gradle/Maven (⌥F12 / Alt + F12)
Este comando abre o terminal integrado ou executa diretamente tarefas configuradas no build tool que você está usando (Gradle ou Maven). Você pode rodar scripts de build personalizados ou usar tarefas específicas, como `build`, `clean`, ou `test`.

### Executar Tarefas de Build Específicas (⌘⇧A / Ctrl + Shift + A)
Este comando abre a ação rápida do IntelliJ para buscar e executar tarefas de build específicas. No caso de Gradle ou Maven, você pode usá-lo para rodar tarefas personalizadas diretamente da IDE.

### Abrir Janela de Build (⌘9 / Alt + 9)
Este atalho abre a janela de build do IntelliJ, onde você pode acompanhar o progresso das compilações e tarefas de build em execução.

## Como Usar Esses Atalhos de Build

Os atalhos de build são especialmente úteis para garantir que o código esteja sempre funcionando e atualizado. Aqui estão algumas dicas para aproveitar ao máximo esses comandos:

1. **Compilar Arquivos Específicos**: Use o atalho de compilar arquivo atual (`⌘F9` / `Ctrl + F9`) frequentemente durante a escrita do código para verificar rapidamente a compilação e encontrar erros antes de tentar um build completo.

2. **Reconstruir Projeto**: Sempre que você fizer alterações substanciais no projeto, como modificar dependências ou alterar a estrutura dos arquivos, utilize o comando de reconstruir projeto para garantir que tudo esteja compilado de maneira limpa.

3. **Gerar Artefatos Regularmente**: Se você está trabalhando em um projeto que será empacotado em um arquivo executável ou biblioteca, como .jar ou .war, o comando de gerar artefatos (`⌥⌘F9` / `Alt + Shift + F9`) será extremamente útil.

4. **Logs de Build**: Não se esqueça de conferir a janela de resultados de build (`⌘0` / `Alt + 0`) para acompanhar logs detalhados e garantir que tudo esteja compilando corretamente sem warnings ou erros ocultos.

Esses atalhos ajudam a melhorar a produtividade ao trabalhar com o processo de build no IntelliJ IDEA, economizando tempo ao compilar e gerar artefatos de maneira eficiente.

# Principais Atalhos para Navegação entre Abas no IntelliJ IDEA

A navegação eficiente entre abas é crucial para a produtividade no desenvolvimento. Aqui estão os principais atalhos para alternar entre as abas abertas no IntelliJ IDEA, incluindo o acesso à aba de projeto e à aba de código.

| **Ação**                                       | **Atalho (macOS)**             | **Atalho (Windows/Linux)**  |
|------------------------------------------------|--------------------------------|-----------------------------|
| **Ir para a Aba Project**                       | `⌘1` (Command + 1)             | `Alt + 1`                   |
| **Ir para a Aba de Código**                     | `⌘2` (Command + 2)             | `Alt + 2`                   |
| **Alternar para a Próxima Aba**                | `⌘⇧]` (Command + Shift + ])    | `Ctrl + Shift + ]`          |
| **Alternar para a Aba Anterior**               | `⌘⇧[` (Command + Shift + [)    | `Ctrl + Shift + [`          |
| **Ir para uma Aba Específica**                 | `⌘[número]` (Command + [número])| `Ctrl + [número]`           |
| **Fechar Aba Atual**                           | `⌘W` (Command + W)             | `Ctrl + F4`                 |
| **Reabrir Última Aba Fechada**                 | `⇧⌘T` (Shift + Command + T)    | `Ctrl + Shift + T`          |
| **Deslocar a Aba para o Lado Esquerdo/Direito**| `⌘Shift + Esquerda/Direita`    | `Ctrl + Shift + Esquerda/Direita` |
| **Mover Aba para Nova Janela**                  | `⌘⇧N` (Command + Shift + N)    | `Ctrl + Shift + N`          |

## Explicação dos Principais Atalhos

### Ir para a Aba Project (⌘1 / Alt + 1)
Esse comando leva você diretamente para a aba **Project**, onde você pode visualizar a estrutura de arquivos do seu projeto, facilitando a navegação e acesso a diferentes arquivos.

### Ir para a Aba de Código (⌘2 / Alt + 2)
Esse comando leva você diretamente para a aba de código atual, permitindo que você volte rapidamente ao editor de código.

### Alternar para a Próxima/Anterior Aba (⌘⇧] / Ctrl + Shift + ])
Esses comandos permitem que você navegue rapidamente entre as abas abertas no editor. Usar `Alternar para a Próxima Aba` (`⌘⇧]` / `Ctrl + Shift + ]`) leva você para a aba seguinte, enquanto `Alternar para a Aba Anterior` (`⌘⇧[` / `Ctrl + Shift + [`) leva você de volta.

### Ir para uma Aba Específica (⌘[número] / Ctrl + [número])
Esse atalho permite que você vá diretamente para uma aba específica. Por exemplo, `⌘3` ou `Ctrl + 3` leva você à terceira aba, e assim por diante, até a décima aba.

### Fechar Aba Atual (⌘W / Ctrl + F4)
Esse atalho fecha a aba que está atualmente em foco, permitindo que você mantenha sua área de trabalho organizada.

### Reabrir Última Aba Fechada (⇧⌘T / Ctrl + Shift + T)
Se você fechou uma aba acidentalmente, esse comando permite que você reabra a última aba fechada, facilitando a recuperação de trabalho anterior.

### Deslocar a Aba para o Lado Esquerdo/Direito
Esses atalhos permitem mover a aba atual para a esquerda ou para a direita entre as abas abertas, ajudando na reorganização do seu espaço de trabalho.

### Mover Aba para Nova Janela (⌘⇧N / Ctrl + Shift + N)
Esse comando permite que você mova a aba atual para uma nova janela, útil para comparar dois arquivos ou focar em um arquivo específico sem distrações.

## Dicas de Uso

- **Use os Atalhos com Frequência**: A prática regular dos atalhos ajuda a torná-los automáticos, aumentando sua eficiência no desenvolvimento.

- **Combine com Outros Atalhos**: Utilize esses atalhos em conjunto com atalhos de build e execução para otimizar seu fluxo de trabalho.

Esses atalhos para navegação entre abas no IntelliJ IDEA são essenciais para manter a fluidez durante o desenvolvimento, permitindo que você alterne rapidamente entre diferentes partes do seu projeto.

# Principais Comandos para Pastas e Arquivos na Aba Project do IntelliJ IDEA

Gerenciar arquivos e pastas de maneira eficiente é crucial para um bom fluxo de trabalho no IntelliJ IDEA. Aqui estão os principais comandos para trabalhar com arquivos e pastas na aba **Project**.

| **Ação**                                      | **Atalho (macOS)**             | **Atalho (Windows/Linux)**  |
|-----------------------------------------------|--------------------------------|-----------------------------|
| **Novo Arquivo**                              | `⌘N` (Command + N)             | `Alt + Insert`              |
| **Novo Pacote**                               | `⌘N` (Command + N)             | `Alt + Insert`              |
| **Novo Diretório**                            | `⌘N` (Command + N)             | `Alt + Insert`              |
| **Renomear Arquivo ou Pasta**                | `F2`                            | `F2`                        |
| **Excluir Arquivo ou Pasta**                 | `⌘Delete` (Command + Delete)   | `Delete`                    |
| **Mover Arquivo ou Pasta**                   | `⌘X` (Command + X)             | `Ctrl + X`                  |
| **Copiar Arquivo ou Pasta**                  | `⌘C` (Command + C)             | `Ctrl + C`                  |
| **Colar Arquivo ou Pasta**                   | `⌘V` (Command + V)             | `Ctrl + V`                  |
| **Mostrar em Finder/Explorer**                | `⌘Shift + R` (Command + Shift + R) | `Ctrl + Shift + R`          |
| **Adicionar ao Controle de Versão**           | `⌘Alt + A` (Command + Option + A) | `Ctrl + Alt + A`            |
| **Localizar Arquivo**                         | `⌘Shift + N` (Command + Shift + N) | `Ctrl + Shift + N`          |
| **Atualizar Estrutura do Projeto**            | `⌘F5` (Command + F5)           | `Ctrl + F5`                 |
| **Criar Nova Classe**                         | `⌘N` (Command + N)             | `Alt + Insert`              |
| **Criar Nova Interface**                      | `⌘N` (Command + N)             | `Alt + Insert`              |
| **Criar Novo Teste**                          | `⌘N` (Command + N)             | `Alt + Insert`              |
| **Mostrar Propriedades do Arquivo**           | `⌘I` (Command + I)             | `Alt + Enter`               |
| **Expandir/Colapsar Todos os Itens**         | `⌘⇧+` (Command + Shift + +)    | `Ctrl + Shift + +`          |
| **Colapsar Tudo**                            | `⌘-` (Command + -)             | `Ctrl + -`                  |
| **Abrir Configurações de Projeto**            | `⌘,` (Command + ,)             | `Ctrl + Alt + S`            |
| **Sincronizar Arquivo com Controle de Versão**| `⌘K` (Command + K)             | `Ctrl + K`                  |
| **Copiar Caminho do Arquivo**                 | `⌘C` seguido de `⌘V` (Command + C, depois Command + V) | `Ctrl + C` seguido de `Ctrl + V` |
| **Visualizar Arquivo em Modo de Edição Rápida** | `⌘E` (Command + E)             | `Ctrl + E`                  |

## Explicação dos Principais Comandos

### Novo Arquivo, Pacote ou Diretório (⌘N / Alt + Insert)
Esse comando permite que você crie rapidamente um novo arquivo, pacote ou diretório dentro da estrutura de projeto. Ao pressionar o atalho, você pode escolher o tipo de item que deseja criar.

### Renomear Arquivo ou Pasta (F2)
Use este comando para renomear rapidamente qualquer arquivo ou pasta selecionada na aba **Project**. É uma maneira eficiente de manter sua estrutura organizada.

### Excluir Arquivo ou Pasta (⌘Delete / Delete)
Este atalho permite que você exclua rapidamente o arquivo ou pasta selecionada. Use com cuidado, pois a exclusão é muitas vezes permanente, a menos que o arquivo esteja em um sistema de controle de versão.

### Mover e Copiar Arquivo ou Pasta (⌘C / ⌘V)
Esses comandos permitem que você copie ou mova arquivos e pastas para outros locais dentro do projeto. Primeiro, selecione o arquivo ou pasta, use `⌘C` para copiar ou `⌘X` para cortar, e depois cole no novo local com `⌘V`.

### Mostrar em Finder/Explorer (⌘Shift + R / Ctrl + Shift + R)
Esse comando abre a localização do arquivo selecionado diretamente no Finder (macOS) ou no Windows Explorer (Windows), facilitando o acesso ao arquivo fora da IDE.

### Adicionar ao Controle de Versão (⌘Alt + A / Ctrl + Alt + A)
Use este atalho para adicionar arquivos ou pastas ao sistema de controle de versão (como Git). Isso é útil para garantir que as alterações sejam rastreadas.

### Localizar Arquivo (⌘Shift + N / Ctrl + Shift + N)
Esse comando abre uma janela para você buscar rapidamente por arquivos no projeto, permitindo que você encontre arquivos sem navegar manualmente pela estrutura de pastas.

### Atualizar Estrutura do Projeto (⌘F5 / Ctrl + F5)
Esse atalho é usado para atualizar a estrutura do projeto, útil quando você fez alterações no sistema de arquivos fora da IDE e precisa que o IntelliJ reconheça essas alterações.

### Criar Nova Classe, Interface ou Teste (⌘N / Alt + Insert)
Com esse comando, você pode rapidamente criar novas classes, interfaces ou arquivos de teste, aumentando a produtividade durante o desenvolvimento.

### Mostrar Propriedades do Arquivo (⌘I / Alt + Enter)
Este comando exibe as propriedades e opções disponíveis para o arquivo selecionado, permitindo que você faça alterações ou visualize informações relevantes.

### Expandir/Colapsar Todos os Itens (⌘⇧+ / Ctrl + Shift + +)
Esses atalhos permitem que você expanda ou colapse todos os itens na aba **Project**, ajudando na organização visual do seu espaço de trabalho.

### Abrir Configurações de Projeto (⌘, / Ctrl + Alt + S)
Use esse comando para acessar rapidamente as configurações do projeto atual, permitindo que você faça ajustes na configuração do ambiente de desenvolvimento.

### Sincronizar Arquivo com Controle de Versão (⌘K / Ctrl + K)
Esse atalho é utilizado para sincronizar as alterações locais com o repositório remoto, essencial para manter seu trabalho atualizado.

### Copiar Caminho do Arquivo (⌘C seguido de ⌘V / Ctrl + C seguido de Ctrl + V)
Esse comando permite que você copie o caminho completo do arquivo selecionado para a área de transferência, facilitando o compartilhamento ou referência em outras partes do seu projeto.

### Visualizar Arquivo em Modo de Edição Rápida (⌘E / Ctrl + E)
Esse comando permite que você visualize rapidamente o arquivo selecionado em um modo de edição leve, facilitando a consulta ou modificação de pequenos trechos de código.

## Dicas de Uso

- **Organize Regularmente**: Utilize os comandos de criação e renomeação frequentemente para manter uma estrutura de projeto organizada.
- **Controle de Versão**: Sempre que adicionar novos arquivos, lembre-se de usar o comando para adicionar ao controle de versão, garantindo que todas as suas alterações sejam rastreadas.
- **Localização Rápida**: Use o comando para mostrar arquivos no Finder/Explorer para acessar rapidamente documentos ou arquivos necessários que estão fora da IDE.

Esses comandos são essenciais para gerenciar arquivos e pastas dentro da aba **Project** do IntelliJ IDEA, melhorando a eficiência e organização do seu fluxo de trabalho.

# Teclas de Atalho do IntelliJ IDEA para Git

O IntelliJ IDEA oferece uma série de atalhos para facilitar o trabalho com Git, tornando o controle de versão mais eficiente. Aqui estão os principais atalhos para operações comuns de Git na IDE.

| **Ação**                               | **Atalho (macOS)**               | **Atalho (Windows/Linux)**  |
|----------------------------------------|----------------------------------|-----------------------------|
| **Fazer Commit**                       | `⌘K` (Command + K)               | `Ctrl + K`                  |
| **Fazer Push**                         | `⌘Shift + K` (Command + Shift + K) | `Ctrl + Shift + K`          |
| **Fazer Pull**                         | `⌘T` (Command + T)               | `Ctrl + T`                  |
| **Mostrar Log do Git**                 | `⌘2` (Command + 2)               | `Alt + 2`                   |
| **Ver Diferenças**                    | `⌘D` (Command + D)               | `Ctrl + D`                  |
| **Abrir Ferramentas de Git**          | `⌘9` (Command + 9)               | `Alt + 9`                   |
| **Exibir Janela de Versão**           | `⌘Alt + K` (Command + Option + K) | `Ctrl + Alt + K`            |
| **Reverter Mudanças**                  | `⌘Z` (Command + Z)               | `Ctrl + Z`                  |
| **Mostrar Alterações no Arquivo**      | `⌘E` (Command + E)               | `Ctrl + E`                  |
| **Trocar de Branch**                   | `⌘⇧A` (Command + Shift + A)      | `Ctrl + Shift + A`          |
| **Criar Nova Branch**                  | `⌘N` (Command + N)               | `Alt + Insert`              |
| **Mesclar Branch**                     | `⌘M` (Command + M)               | `Ctrl + M`                  |
| **Exibir Histórico de Commits**        | `⌘6` (Command + 6)               | `Alt + 6`                   |

## Explicação dos Principais Atalhos

### Fazer Commit (⌘K / Ctrl + K)
Abre a janela de commit, permitindo que você insira uma mensagem e selecione os arquivos a serem commitados.

### Fazer Push (⌘Shift + K / Ctrl + Shift + K)
Envia seus commits locais para o repositório remoto.

### Fazer Pull (⌘T / Ctrl + T)
Atualiza sua branch local com as últimas mudanças do repositório remoto.

### Mostrar Log do Git (⌘2 / Alt + 2)
Abre a janela de log do Git, onde você pode visualizar o histórico de commits.

### Ver Diferenças (⌘D / Ctrl + D)
Mostra as diferenças entre a versão atual do arquivo e a versão no repositório.

### Abrir Ferramentas de Git (⌘9 / Alt + 9)
Acessa rapidamente as ferramentas do Git dentro do IntelliJ.

### Exibir Janela de Versão (⌘Alt + K / Ctrl + Alt + K)
Mostra a janela de diferenças e revisões do Git para o arquivo atual.

### Reverter Mudanças (⌘Z / Ctrl + Z)
Reverte a última mudança feita no arquivo atual.

### Mostrar Alterações no Arquivo (⌘E / Ctrl + E)
Mostra as alterações que foram feitas no arquivo atual em comparação com a última versão commitada.

### Trocar de Branch (⌘⇧A / Ctrl + Shift + A)
Permite que você mude rapidamente entre branches do seu repositório.

### Criar Nova Branch (⌘N / Alt + Insert)
Inicia o processo para criar uma nova branch no repositório.

### Mesclar Branch (⌘M / Ctrl + M)
Abre a opção para mesclar uma branch na branch atual.

### Exibir Histórico de Commits (⌘6 / Alt + 6)
Acesso rápido ao histórico de commits do seu projeto, permitindo revisar alterações anteriores.

## Dicas de Uso

- **Personalização de Atalhos**: Você pode personalizar os atalhos no IntelliJ indo em `Preferences` > `Keymap`.

- **Prática Regular**: Use os atalhos regularmente para se acostumar e melhorar sua eficiência.

- **Documentação**: Consulte a documentação do IntelliJ IDEA para mais detalhes e opções.

Esses atalhos do Git no IntelliJ IDEA ajudam a tornar o controle de versão mais eficiente, permitindo que você gerencie seu código de forma eficaz.
